    笔记5：训练神经网络I

    激活函数¶

    • Sigmoid： (\(\frac{1}{1+e^{-x}}\)\) 特点：
      • “饱和”现象会让梯度消失
      • 不以 0 中心：这样导致下一层神经元的输入偏向于正值，假设中间某层神经元有 \(\(\text{Output} = f(\sum_i w_i x_i+b)\)\) 其中 \(f\) 为激活函数，则该层参数 \(w_i\) 梯度为 \(\(\begin{aligned}\frac{\partial L}{\partial w_i} &= \frac{\partial L}{\partial f} \frac{\partial f}{\partial (\sum_i w_i x_i+b)} \frac{\partial (\sum_i w_i x_i+b)}{\partial w_i} \\&= \frac{\partial L}{\partial f} \frac{\partial f}{\partial (\sum_i w_i x_i+b)} x_i\end{aligned}\)\) 注意到 \(\frac{\partial f}{\partial (\sum_i w_i x_i+b)}\) 和 \(x_i\) 均恒为正数，故其 \(w_i\) 的梯度始终与前面的梯度 \(\frac{\partial L}{\partial f}\) 的符号相同。假设该神经元中有两个参数 \(w_0, w_1\) 都希望达到最优解，而最优解一个比 \(w_0\) 大，一个比 \(w_1\) 小，而两者只能向同一方向移动，也就会导致学习路线曲折，收敛速度慢：
      • 指数函数计算开销较大
    • tanh：以 0 为中心，但仍然会饱和
    • ReLU：
      • 优点：不饱和、计算高效、收敛速度快、生理学上合理
      • 缺点：不以 0 为中心、小于 0 时依然有梯度消失的问题（坏死现象：如果权重初始得不好或者学习率过大，导致某个神经元传给 ReLU 的值小于 0，进而梯度为 0，无法更新参数，神经元坏死）
      • 解决方法：设置偏置项，或使用 Leaky ReLU、ELU 等

    总结： - 使用 ReLU，设置好的学习率 - 使用 Leaky ReLU 或 ELU 或 Maxout - 使用 tanh（但这大概率不好） - 不要用 Sigmoid

    数据预处理¶

    1. 数据处理：一般处理至以 0 为中心，而不需要归一化，因为图像像素值都处于差不多的范围内。当然也不会做 PCA 之类的处理，这是为了保留图像的原结构。
    2. 在测试和训练时都会减去中心图像。在训练开始前进行数据预处理操作（也可以采样得到均值）。
    3. 两种方法：
      1. 图像去均值：先求出中心图像 image，这个图像是 \(H\times W\times 3\) 的，然后每个图像减去中心图像。
      2. 通道去均值：先求出每个通道的均值，得到三个数，然后每个图像的每个通道的每个像素都减去对应通道的值即可。

    权重初始化¶

    • 不能全 0，不能全相同值（参数对称问题）
    • 不能全部初始化为小的随机数（激活值逐渐消失 \(\rightarrow\) 梯度消失）
    • 也不能全部很大，否则会饱和

    结论：

    • 使用 tanh 作为激活函数时，使用 Xavier 初始化——从标准高斯取样，并根据输入数量进行缩放；使用 ReLU 作为激活函数，则还需额外除 2（因为一半的神经元失活）
    • 核心：输入数据方差=输出数据方差
    • 如何初始化权重问题依然活跃，一般从 Xavier 开始尝试，然后再尝试其他方法

    监督学习过程¶

    1. 数据预处理
    2. 选择神经网络结构
    3. 初始化网络：
      • 完整性检查
        • 检查损失函数的初始值
        • 加入正则项，再检查损失函数
    4. 开始训练：
      1. 从小的数据集开始，关闭正则项，确保 loss 可能降至接近 0
      2. 开启正则项，寻找合适的学习率：
        • 如果 loss 基本不变 \(\rightarrow\) 学习率太低（这里可能 acc 突变，因为可能突然选中正确的，例如各分类的 Sigmoid 值为 0.11，0.1，0.1，0.1，...，0.1，损失函数基本不变，但 acc升高）
        • 如果 loss 到了 NaN \(\rightarrow\) 学习率太高
        • 一般设置在 1e-3 ~ 1e-5 之间
